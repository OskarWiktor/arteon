## Zaawansowane wzorce AI (z humanizer)

| Wzorzec                | Problem                       | Fix                                     |
| ---------------------- | ----------------------------- | --------------------------------------- |
| **Rule of Three**      | "innowacja, jakość, sukces"   | Opisz konkretnie, nie listuj abstrakcji |
| **Em dash overuse**    | zbyt wiele — myślników        | Użyj przecinków lub kropek              |
| **Elegant variation**  | ciągłe synonimy jednego słowa | Powtórzenia są OK                       |
| **False ranges**       | "od X do Y" bez skali         | Usuń lub podaj konkretne liczby         |
| **Sycophantic tone**   | "Świetne pytanie!"            | Przejdź do sedna                        |
| **Inflated symbolism** | "This represents..."          | Opisz bez nadawania znaczenia           |

## Tekst "bez duszy" (soulless writing)

Nawet "czysty" tekst może być oczywisto AI jeśli:

- Każde zdanie ma tę samą długość
- Brak opinii, tylko neutralne raportowanie
- Brak niepewności lub mieszanych uczuć
- Brak perspektywy pierwszej osoby
- Brak humoru, charakteru, osobowości

**Jak dodać głos:**

- Miej opinię ("Zauważyliśmy, że...")
- Mieszaj rytm zdań (krótkie. Potem dłuższe, które budują myśl.)
- Przyznaj złożoność ("To skuteczne, ale wymaga czasu")
- Używaj "my" gdy pasuje

# CZĘŚĆ 3: WRITING RULES

## 1. Zacznij od sedna

```
❌ "W dzisiejszych czasach strona internetowa jest niezbędna..."
✅ "Strona internetowa pomaga pozyskiwać klientów na trzy sposoby..."

❌ "Wielu przedsiębiorców zastanawia się..."
✅ "Dobra strona generuje leady. Oto jak to działa..."
```

## 2. Wyjaśniaj terminy natychmiast

```
❌ "Optymalizacja Core Web Vitals jest ważna."
✅ "Core Web Vitals to metryki Google mierzące szybkość i wygodę
   korzystania ze strony. Dobre wyniki pomagają w pozycjonowaniu."

❌ "Potrzebujesz PWA."
✅ "PWA (Progressive Web App) to strona, która działa jak aplikacja
   mobilna — bez instalacji ze sklepu."
```

## 3. Pełne zdania (nie równoważniki)

```
❌ "Szybka realizacja. Konkurencyjne ceny. Doświadczony zespół."
✅ "Realizujemy projekty w 2-4 tygodnie. Nasze ceny są konkurencyjne
   w stosunku do jakości. Zespół ma 10 lat doświadczenia."
```

## 4. Konkretne przykłady

```
❌ "Pomagamy firmom z różnych branż."
✅ "Pracowaliśmy z kancelariami prawnymi, klinikami medycznymi,
   restauracjami i sklepami e-commerce."
```

## 5. Unikaj generalizacji

```
❌ "SEO jest ważne dla każdej firmy."
✅ "SEO pomaga klientom znaleźć Twoją firmę w Google.
   Dla lokalnych usług to często główne źródło nowych klientów."
```

# CZĘŚĆ 8: FACT VERIFICATION

## All Claims Must Be True

```
❌ "SEO zwiększa ruch o 300%"
   (bez źródła, generalizacja)

✅ "Według badania Ahrefs z 2023, strony na pozycji 1
   otrzymują średnio 27.6% wszystkich kliknięć"
   (źródło, konkretne dane)
```

## Statistics Need Sources

```markdown
✅ Core Web Vitals wpływa na rankingi — Google potwierdziło
to w [oficjalnym ogłoszeniu](link).

❌ 90% użytkowników opuszcza wolne strony.
(skąd ta liczba?)
```

## Hypotheticals Must Be Marked

```
✅ "Załóżmy, że prowadzisz restaurację..."
✅ "Przykład: Firma X zwiększyła sprzedaż o 20%"

❌ "Firmy zwiększają sprzedaż o 20% dzięki SEO"
   (generalizacja bez dowodu)
```

## Common Issues

| Issue        | Bad                      | Good                           |
| ------------ | ------------------------ | ------------------------------ |
| Vague claims | "Najlepszy generator"    | "Generator z 9 harmoniami"     |
| Outdated     | "Google używa FID"       | "Google używa INP od 2024"     |
| Unverifiable | "80% firm nie ma strony" | "Według GUS 2023, 75% firm..." |

## Acceptable Sources

- Oficjalne blogi (Google, React)
- Badania (Ahrefs, Moz, Semrush)
- Raporty branżowe
- Dokumentacja techniczna
- Statystyki rządowe (GUS)

## Unacceptable Sources

- Wikipedia (secondary)
- Fora internetowe
- Nieweryfikowalne źródła
- Bardzo stare dane (>3 lata)

# CZĘŚĆ 9: WORKFLOW

## Before Starting Any Content Task

```
1. Check docs/TASKS.md for task details
2. Read docs/CONTENT_INSTRUCTIONS.md
3. Review relevant CATALOG (BLOG, TOOLS, COMPONENTS)
4. Research keywords (if SEO content)
5. Review similar existing content
```

## Content Creation Process

```
1. Write content following these guidelines
2. Self-review against quality checklist
3. Update JSON files
4. Validate JSON syntax
5. Present changes in table format
```

## After Completion

```
1. Present changes in table format
2. Update TASKS.md status
3. Move completed task to DONE_TASKS.md
4. Build test (if required)
```

## Build Requirements

| Scenario                                      | Build Required? |
| --------------------------------------------- | --------------- |
| Text-only changes                             | ❌ No           |
| Content edits in JSON                         | ❌ No           |
| Tasks marked "Weryfikacja: nie jest wymagana" | ❌ No           |
| Code changes                                  | ✅ Yes          |
| UI modifications                              | ✅ Yes          |
| Logic changes                                 | ✅ Yes          |

# CZĘŚĆ 11: QUALITY CHECKLISTS

## Content Quality (przed publikacją)

- [ ] Wstęp zaczyna od sedna (nie "W dzisiejszych czasach...")
- [ ] Każdy termin techniczny wyjaśniony natychmiast
- [ ] Brak fraz AI-typowych (Rule of Three, em dash overuse)
- [ ] Brak korpo-języka
- [ ] Brak anglicyzmów (lub wyjaśnione)
- [ ] Pełne zdania (nie równoważniki)
- [ ] Konkretne przykłady z właściwych branż
- [ ] Mieszany rytm zdań (nie wszystkie tej samej długości)

## SEO Quality

- [ ] Meta title 50-60 znaków, keyword na początku
- [ ] Meta description 150-160 znaków
- [ ] Bez "online" w title/description
- [ ] Keyword w H1
- [ ] Internal links do usług/narzędzi
- [ ] HowTo schema dla tools
- [ ] alternateName w schema

## Fact Verification

- [ ] Każde zdanie jest prawdziwe
- [ ] Każda liczba/statystyka ma źródło
- [ ] Źródła są aktualne (max 2-3 lata)
- [ ] Linki działają
- [ ] Przykłady hipotetyczne oznaczone
- [ ] Brak niepotwierdzonych claimów

## Technical

- [ ] JSON syntax valid
- [ ] All locales updated (if applicable)
- [ ] Images have alt text
- [ ] No broken links
